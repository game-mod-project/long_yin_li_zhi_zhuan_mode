## Smoke 2차 회귀 + UX 변경 — 4 iteration 으로 해결

본 release 의 v0.7.3 strip 회귀 학습 (HANDOFF §4.3) 덕분에 IL2CPP IMGUI strip 회귀 0건. 다른 회귀·UX 4 회 발견·수정.

### 회귀 / UX 1차 (`255ffef`): focus stale 감지 강화 (raw ref equality)

smoke 1차 발견:
- `focus = (Inventory, idx=5)` 상태에서 이동 후 IL2Cpp List packing → idx 5 가 within bounds 여도 다른 item 으로 바뀌어 panel 이 stale 표시
- 사용자: "이동 복사 후 fucus는 해제 될때도 있도 아닐때도 있음"

**Fix**: SetFocus 시점에 raw item reference 캡처 (`_focusRawRef`). Set*Rows 호출 시 `ReferenceEquals(rawItems[idx], _focusRawRef)` 검증 — mismatch 시 focus auto-clear.

### 회귀 / UX 2차 (`2fecd91`): DrawItemList click handler SetFocus 호출

1차 fix 후 smoke 결과:
- 사용자: "상세 팝업 오픈전 -> 셀 클릭 (외곽선 표시 출력). 상세 팝업 오픈후 -> 셀클릭 동작 안함"
- 원인: DrawItemList 의 cell click handler 가 `_focus = (area, idx)` 직접 할당 → `_focusRawRef` 안 갱신. ItemDetailPanel 열린 후 `GetFocusedRawItem` 의 ref equality 검증이 매 frame focus clear

**Fix**: 직접 할당을 `SetFocus(area, idx)` public method 호출로 교체 — `_focus + _focusRawRef` 동시 갱신.

### UX 3차 (`f191e23`): Cell 클릭 = single-select

smoke 1·2·3 동작 확인 후 사용자 요청:
- "셀 클릭과 아이템 선택이 구분되어 있는데 셀 클릭시 -> 해당 아이템도 같이 선택, 이 경우는 무조건 한개씩만 (클릭된 셀에 해당하는 아이템만 체크). 아이템 클릭시 -> 셀은 선택되지 않음, 이 경우 기존 동작과 동일"

**Fix**: cell click 시 같은 area 의 `checks.Clear()` + `checks.Add(r.Index)` — single-select 진입점. Toggle 라벨 multi-check 와 워크플로우 분리.

### 회귀 / UX 4차 (`f46ef1f`): Toggle multi-check 시 cell focus 해제 + 컨테이너 area focus visual

3차 fix 후 사용자 보고:
- "아이템 클릭하여 복수 선택시 Cell focus 해제하기 (인벤,창고, 컨테이너 동일)"
- "인벤토리와 창고에서는 동작하지만 컨테이너는 상세보기 팝업 상태에서 셀 클릭 동작하지 않음"

**Fix 1 (Toggle)**: Toggle 라벨 click 시 `now != was` 발생하면 같은 area 의 cell focus 해제. multi-check 워크플로우 진입 = cell focus 종료 신호.

**Fix 2 (컨테이너)**: `_containerRawItems` 가 의도적으로 빈 list (JSON path — ItemDetailReflector 미지원) 인데, `GetFocusedRawItem` 의 OOB check 가 매 frame focus auto-clear → 사용자에게 "cell 클릭 동작 안함" 으로 보임.

→ `GetFocusedRawItem` 와 `InvalidateFocusIfStale` 모두 raw list 가 빈 경우 ref check skip + focus 유지. panel 데이터는 null 반환 (DrawEmpty), outline 만 시각 피드백 — 분리된 책임 (visual ≠ panel content).
