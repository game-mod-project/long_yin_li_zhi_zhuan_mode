## Phase 3 충돌 디버깅 narrative

**1차 시도** (commit `f950d40`):
- HeroDataCapBypassPatch 4 Postfix 등록 + ConfigEntry 4 추가
- 사용자 smoke 보고: `EnableUncapMax = true` + 자질값 999 [저장] → toast 성공이지만 row 즉시 120 reset

**원인 진단** (LogOutput.log 분석):
```
[Info :LongYin Roster Mod] HeroDataCapBypassPatch: HeroData.RefreshMaxAttriAndSkill patched (Prefix+Postfix)
[Info :LongYin InGame Cheat] [Harmony] 패치됨: HeroData.RefreshMaxAttriAndSkill   ← 충돌
```

LongYinCheat 가 같은 method 들을 patch. cheat `MultiplierPatch.RefreshMaxPostfix` (line 312-316) 가 자체 EnableUncapMax=false 일 때 `ClampMaxStats(maxAttri, 120f)` 강제 호출. 우리 mod 와 cheat 의 patch 등록 순서:
- 우리 등록 (EARLY) → cheat 등록 (LATE) → Harmony queue 순서로 호출 → cheat 가 마지막 = clamp 120 win

**2차 fix** (commit `520f121`):
- HarmonyMethod priority 설정:
  - Postfix = `Priority.Last` (=0) → cheat 보다 LATE 실행, 마지막에 999 복원 win
  - Prefix = `Priority.First` → cheat 보다 EARLY 실행, snapshot 먼저
- 4 patches 모두 적용

**결과**: 사용자 재smoke → 자질값 999 유지 PASS.
