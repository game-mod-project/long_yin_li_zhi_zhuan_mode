## IL2CPP strip 회귀 — 2 iteration 으로 해결

본 release 의 spec §6 가 strip 위험을 사전 예측했고, 실제 smoke 에서 발견됨. 두 번 fallback 적용 후 PASS.

### 1차 strip (commit `2fa8127` — 초기 plan 구현)

```
[Warning] ContainerPanel.Draw threw: NotSupportedException: Method unstripping failed
[Error] GUI Error: You are pushing more GUIClips than you are popping.
```

매 frame 폭주. 화면에 toolbar 만 살고 row 영역 비어 있음.

**원인**: `GUILayout.Box(string, params GUILayoutOption[])` 와 `GUI.Box(Rect, string)` 가 IL2CPP 빌드에서 strip.

**1차 fallback** (commit `0a2d4c5`):
- `GUILayout.Box(...)` → `GUILayout.Label("", GUILayout.Width, GUILayout.Height)` (SlotDetailPanel.cs:152 검증)
- `GUI.Box(Rect, "")` → `GUI.DrawTexture(rect, Texture2D.whiteTexture)` (DialogStyle.FillBackground 검증)

→ **여전히 같은 오류 폭주**. Box 류는 fallback 됐으나 다른 method 가 strip.

### 2차 strip 발견 (commit `0a2d4c5` 의 fallback 이후)

같은 `Method unstripping failed` 폭주 지속. 의심 좁혀가기:
- `GUI.DrawTexture` ✓ (DialogStyle.FillBackground 작동 — 검증됨)
- `Texture2D.whiteTexture` ✓ (DialogStyle 사용 중)
- `GUI.color` ✓ (everywhere)
- `GUI.Label(Rect, string)` ✓ (DialogStyle.DrawHeader 사용)
- `GUILayout.Label(string, options)` ✓ (SlotDetailPanel.cs:152 사용)
- **`GUILayoutUtility.GetLastRect()`** ❌ — grep 결과 v0.7.2 어디에서도 미사용. plan §4.5 의 "v0.7.2 / v0.7.0.1 에서 비슷한 패턴 작동 중 (검증됨)" 주장 오류.

**2차 fallback** (commit `db0303a`):
- `GUILayout.Label("", options)` + `GUILayoutUtility.GetLastRect()` 2-call → `GUILayoutUtility.GetRect(w, h, options)` 1-call

→ **PASS**. GetRect 는 strip 대상 아니었음.

## 자산 / 회귀 방지

### plan/spec 공정 학습
- spec §6 "GUI.Box strip 위험" 은 정확했으나 fallback 검증 부족 (실제 fallback 도 strip 됨 — 2차 fallback 필요)
- plan §4.5 의 "GetLastRect v0.7.2 검증" 주장은 grep 으로 확인 시 거짓이었음 — plan 단계에서 검증하지 않은 추정
- 향후 IMGUI patterns 추가 시 "v0.7.2 production code 의 정확한 사용처 grep" 이 spec/plan 단계 필수 체크리스트

### IL2CPP strip-safe 검증된 IMGUI patterns (v0.7.3 시점)
- `GUI.color` getter/setter
- `GUI.DrawTexture(Rect, Texture)` 2-arg
- `GUI.Label(Rect, string)` 2-arg
- `GUI.Box(Rect, string)` 2-arg — ❌ strip
- `GUI.Button(Rect, string)`
- `GUI.Window(int, Rect, WindowFunction, string)`
- `GUI.DragWindow(Rect)`
- `GUILayout.Label(string)` 1-arg
- `GUILayout.Label(string, params GUILayoutOption[])` ✓ (SlotDetailPanel)
- `GUILayout.Box(string, params GUILayoutOption[])` — ❌ strip
- `GUILayout.Button(string, params)`
- `GUILayout.TextField(string, params)`
- `GUILayout.Toggle(bool, string, params)`
- `GUILayout.Space(float)`
- `GUILayout.BeginHorizontal/EndHorizontal/BeginVertical/EndVertical/BeginScrollView/EndScrollView`
- `GUILayout.Width(float)`, `GUILayout.Height(float)`
- `GUILayoutUtility.GetLastRect()` — ❌ strip
- **`GUILayoutUtility.GetRect(float, float, params GUILayoutOption[])`** ✓ (v0.7.3 검증)
- `Texture2D.whiteTexture` static property
- `GUI.skin.label.alignment / fontSize` (DialogStyle.DrawHeader)

### 폐기 (strip 으로 사용 금지)
- `GUILayout.Box(string, ...)` — 자리 잡기 + 색칠 동시 필요 시 `GUILayout.Label("", options)` + `GUI.DrawTexture` 조합 또는 `GUILayoutUtility.GetRect` 직접 호출
- `GUI.Box(Rect, string)` — `GUI.DrawTexture(rect, Texture2D.whiteTexture)` + `GUI.color` 로 대체
- `GUILayoutUtility.GetLastRect()` — `GUILayoutUtility.GetRect(w, h, options)` 로 대체 (1-call API)
- `new GUIStyle(GUIStyle)` ctor — 사용 금지 (HANDOFF §4.3 기존 항목)
- `GUILayout.FlexibleSpace()` — 사용 금지 (v0.7.0.1 기존 항목)
