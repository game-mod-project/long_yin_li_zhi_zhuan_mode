## 7. iteration fix narrative

### 7.1 Test compile error: `KeyCode` 미정의

**증상**: `dotnet test` 실행 시 `CS0246: KeyCode` not found.

**원인**: `LongYinRoster.Tests` 프로젝트가 UnityEngine 직접 reference 안 함 — `UnityStubs.cs` 가 stub 제공. `KeyCode` enum + `Input` 클래스 + `Event.keyCode` 누락.

**수정**: `UnityStubs.cs` 에 `KeyCode` enum (subset: None / Backspace / Tab / Return / Escape / Space / Alpha0~9 / A~Z / Keypad0~9 / F1~F12 / shift·ctrl·alt) + `Input` static stub (모든 method false) + `Event.keyCode` field 추가.

### 7.2 Test compile error: `Logger` 모호성 (Release build only)

**증상**: `CS0104: 'Logger' is an ambiguous reference between 'LongYinRoster.Util.Logger' and 'UnityEngine.Logger'`. Test 빌드는 통과 (UnityStubs 에 UnityEngine.Logger 없음), Release 빌드만 실패.

**원인**: SettingsPanel.cs 가 `using LongYinRoster.Util;` + `using UnityEngine;` 동시 사용. UnityEngine.Logger (실 Unity 빌드 시 존재) 와 LongYinRoster.Util.Logger 동일 이름.

**수정**: `using Logger = LongYinRoster.Util.Logger;` alias 추가 (ModWindow.cs 의 기존 패턴 mirror).

### 7.3 _hydrated 필드 unused warning (TreatWarningsAsErrors)

**증상**: `CS0414: '_hydrated' assigned but never used` (Task 2 skeleton 단계).

**원인**: Task 2 placeholder OnGUI 가 `_hydrated` 미사용. Task 3 UI 작성 후 `if (!_hydrated) Hydrate();` 추가로 자동 해소.

**교훈**: skeleton 단계에서 미사용 필드는 Task 3 까지 도달 전이면 placeholder 라도 reference 추가 필요 (TreatWarningsAsErrors 환경).

총 3 iteration fix — Tests/Release 빌드 + UnityStubs 갱신 + alias 추가.
