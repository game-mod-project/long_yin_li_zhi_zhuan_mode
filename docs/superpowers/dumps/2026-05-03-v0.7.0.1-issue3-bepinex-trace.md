## 핵심 진단

BepInEx LogOutput.log 에 두 가지 에러가 매 frame (수천 회) 무한 반복:

### 에러 1 — DrawToast strip
```
[Error  :Il2CppInterop] Exception in IL2CPP-to-Managed trampoline:
  System.NotSupportedException: Method unstripping failed
   at LongYinRoster.UI.ContainerPanel.DrawToast()
   at LongYinRoster.UI.ContainerPanel.Draw(Int32 id)
```

→ **`GUILayout.FlexibleSpace()` IL2CPP strip** (HANDOFF §4.3 명시). spec §3.3 의 root cause 1순위 가설 확정.
→ Phase 2 Task 2.2 의 DrawToast no-op 로 fix 예정.

### 에러 2 — Draw 안의 GUILayout.Space(Single) strip (예상 외)
```
[Error  :Il2CppInterop] Exception in IL2CPP-to-Managed trampoline:
  Il2CppInterop.Runtime.Il2CppException:
  System.ArgumentException: Getting control 0's position in a group with only 0 controls when doing repaint
   at UnityEngine.GUILayoutGroup.GetNext ()
   at UnityEngine.GUILayoutUtility.DoGetRect (...)
   at UnityEngine.GUILayout.Space (System.Single pixels)
   at LongYinRoster.UI.ContainerPanel.Draw(Int32 id)
```

→ **`GUILayout.Space(Single)` 도 IL2CPP strip 가능 — HANDOFF §4.3 가정과 모순**.

ContainerPanel.cs 의 `Space` 호출 위치:
- line 114: `GUILayout.Space(DialogStyle.HeaderHeight);`
- line 117: `GUILayout.Space(4);`
- line 153: `GUILayout.Space(4);`

ModeSelector.cs:46 의 `GUILayout.Space(DialogStyle.HeaderHeight + 4);` 는 정상 동작 (사용자가 F11+2 메뉴 정상 표시 — F11 → 모드 메뉴 진입 가능).

**가설**: DrawToast 의 FlexibleSpace strip fail 이 IL2CPP-Interop 의 internal trampoline cache 를 corrupted state 로 만들어 같은 frame/thread 의 후속 `GUILayout.Space` 호출도 cascade fail. 즉 root cause 는 1개 (FlexibleSpace), DrawToast no-op 후 자동 해결될 가능성이 높음.

## Phase 2 Implementation 가이드

### Primary fix (1순위 가설 검증)
1. `ContainerPanel.Toast` → `ToastService.Push` 로 위임
2. `ContainerPanel.DrawToast` → no-op (FlexibleSpace 제거)
3. `ContainerPanel.OnGUI` 외곽 try/catch 가드
4. `ContainerPanel.Draw` 외곽 try/catch 가드

→ Primary fix 만으로 cascade fail 끊어지면 Space 호출은 그대로 유지 가능 (사용자 인게임 재검증 으로 확인).

### Fallback (Primary 후도 Space strip 잔존 시)
사용자 인게임 재검증에서 동상 에러가 나면:
- `ContainerPanel.cs:114` 의 `GUILayout.Space(DialogStyle.HeaderHeight)` → `GUILayout.Label("", GUILayout.Height(DialogStyle.HeaderHeight))`
- `ContainerPanel.cs:117, 153` 의 `GUILayout.Space(4)` → `GUILayout.Label("", GUILayout.Height(4))`

또는 모든 `GUILayout.Space(N)` 호출을 빈 라벨로 일괄 교체.
