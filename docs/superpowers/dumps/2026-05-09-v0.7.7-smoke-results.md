## 3. 사용자 검증 사이클 — 5 iteration fix narrative

### Iteration 1: rareLv/itemLv row 안 보임
**증상**: 사용자 보고 — "상세 팝업에 등급(rareLv)/품질(itemLv) 변경 안 보임. 강화 row 등 수정 안 됨".
**원인**: ItemDetailReflector.GetCuratedFields 의 conditional logic (`if (enh > 0) result.Add(...)`) 가 enhanceLv=0 일 때 row 자체 미표시 + rareLv/itemLv 자체가 curated 에 등재 안 됨.
**Fix**: edit mode 시 ItemDetailReflector curated 무시, **ItemEditFieldMatrix.ForCategory 직접 렌더** (모든 edit-able 필드 항상 표시).

### Iteration 2: 강화/특수강화 단순 수치 수정 무의미
**증상**: 사용자 피드백 — "강화/특수 강화는 단순 수치 수정 의미 없음. 실제 base/additional stat 추가가 필요".
**원인**: 게임 강화 메커니즘이 stat 추가 시스템. enhanceLv=N 만 변경해도 baseAddData/extraAddData 가 함께 변경 안 되면 효과 없음.
**Fix**: ItemEditFieldMatrix Equipment 에서 enhanceLv/speEnhanceLv 제거 (speWeightLv 유지). 대신 **HeroSpeAddData stat editor** 신규 도입.

### Iteration 3: HeroSpeAddData 구조 spike v1 부족
**증상**: F8 spike v1 — `rareLv field not found` + `equipmentData null` + `RefreshSelfState false`.
**원인**:
- `GetField(name)` 만 사용 → Property 검색 안 함
- type 도 Property 였음 → spike 의 type=0 매칭 실패 → first item 잡음 (Equipment 아닐 수 있음)
- RefreshSelfState 가 정확한 method 이름 아님

**Fix**: spike v2 — Property 도 검색 + 모든 카테고리 wrapper sample 확보 + Refresh* method 모두 dump + HeroSpeAddData 내부 구조 dump.

### Iteration 4: SpeAddType 55+ "기타" 표시
**증상**: 사용자 보고 — "55+ 인게임 한글 표시되는데 mod 는 '기타'".
**원인**: SpeAddTypeNames 가 0~54 만 cover (LongYinCheat reference dump 의 "..." 부분 미수록).
**Fix**: LongYinCheat 디컴파일 cache (`C:/Users/deepe/AppData/Local/Temp/longyincheat_decomp/CharacterFeature.cs:94-230`) 의 풀 dump 추출 → **134 entry** (idx 0~207, 일부 sparse). 신규 strip-safe (`Get/AllOrdered`).

### Iteration 5: dropdown UI 작은 panel 에서 안 맞음 + 등급/품질도 selector 요청
**증상**: 사용자 피드백 — "panel 작아 dropdown 보기 힘듦. 등급/품질도 selector 방식".
**Fix**:
- panel 크기 380×500 → **480×640** (+ Init 의 `Math.Max` clamp 로 기존 사용자 cfg auto-bump)
- inline dropdown 제거 → 신규 **`SelectorDialog`** modal popup window (검색 box + scrollable, 360×480)
- rareLv/itemLv 도 textfield → selector 버튼
- 라벨 swap fix: itemLv=등급, rareLv=품질 (ItemReflector.cs:20 주석 기준)
- ItemRareLvNames 추가 — EquipLvNames (열악/보통/우수/정량/완벽/절세) + QualityNames (잔품/하품/중품/상품/진품/극품)
