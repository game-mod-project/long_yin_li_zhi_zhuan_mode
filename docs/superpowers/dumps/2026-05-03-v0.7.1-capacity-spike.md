## 핵심 발견

1. ItemListData 의 capacity 키워드 매칭 property/field 는 **`maxWeight` (Single/float) 단 하나**
2. 갯수 기반 capacity 자체가 게임에 없음. **인벤·창고 모두 무게 기반**
3. 사용자 보고된 "인벤 over-cap 허용 (속도 페널티) / 창고 hard cap" 의 capacity 는 **무게**
4. HANDOFF 의 `171 / 217` 은 dump 시점의 현재 갯수 — capacity 가 아님

## Spec 영향 (사용자 (B) 확정)

**하이브리드 표시** — 갯수+무게 둘 다 노출, 가드는 무게 기반:
- 라벨: `"인벤토리 (45개, 720.3 / 964 kg)"`, `"창고 (32개, 156.8 / 300 kg)"`
- over-cap 마커 (인벤만): `"인벤토리 (180개, 1020.5 / 964 kg) ⚠ 초과"`
- 가드: 무게 기반 — `available_weight = max(0, maxWeight - currentWeight)`, 추가 시도 item.weight 누적해서 초과 시 Failed
