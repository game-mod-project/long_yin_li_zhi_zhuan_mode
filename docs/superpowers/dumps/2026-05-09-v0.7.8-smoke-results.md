## 3. 사용자 7 iteration fix narrative

### Iteration 1: maxhp/maxpower/maxmana = 0 표시 + Quick 후 0 으로 떨어짐
**원인**: HeroData property 명이 camelCase (maxhp/maxMana/maxPower). 내 매트릭스가 모두 lowercase. + cheat StatEditor 검증 — `realMaxHp/realMaxMana/realMaxPower` 동기화 필수.
**Fix**: PlayerEditApplier `GameSelfMethods` / `RealMaxMirror` dict 도입. ApplyResource 4.5 step (max 변경 시 realMax 도 set). FakePlayerNoMethod 의 mock field 도 camelCase + realMax mirror.

### Iteration 2: Quick 회복 vs 채움 통합 / max 표시 무의미 / current 값 max 검증
**Fix**: QuickFullHeal = hp+mana+power 통합 (QuickRestoreEnergy 는 obsolete redirect). DrawResourceRow 의 `/{max}` 표시 제거. CurrentToMax dict — hp/mana/power 입력 시 maxhp/maxMana/maxPower 로 자동 clamp.

### Iteration 3: 무공 lv 모두 0 표시
**원인**: cheat SkillManager.cs:315 검증 — `KungfuSkillLvData.lv` (short form). 내 spike 가 `level` 검색해서 read 실패.
**Fix**: KungfuSkillEditor.GetEntries `ReadInt(entry, "lv")`.

### Iteration 4: SpeAddType 55+ "기타" 표시

**Fix**: LongYinCheat 디컴파일 cache (CharacterFeature.cs:94-230) 풀 dump 추출 → 134 entry (idx 0~207).

### Iteration 5: dropdown UI 작은 panel 안 안 맞음 / 등급-품질도 selector
**Fix**: SelectorDialog modal popup. 등급/품질/속성 모두 selector. itemLv=등급 / rareLv=품질 라벨 swap (ItemReflector.cs:20 검증). ItemRareLvNames + EquipLvOptions/QualityOptions.

### Iteration 6: 강화/특수강화 단순 수치 무의미 → 돌파속성 도입
