## 천부 중복 추가 fix (v0.5.1 release 직전 추가)

**User reported**: 천부 덮어쓸 때 이미 존재하는 영구 천부가 다시 추가되어 중복으로 가지게 되는 문제.

**Root cause**: `ClearAllTempTag` 가 영구 (isPermanent=true) 천부는 clear 안 하므로 slot JSON 의 영구 tag 가 player 에 이미 있으면 `AddTag` 호출 시 중복 추가됨.

**Fix**: clear 후 시점의 `player.heroTagData` 의 tagID 를 수집해 already-exists 검사 → 있으면 skip + log `heroTag[ID] (이미 존재 — 중복 방지)`.

**검증**: ✅ **PASS** (사용자 보고). 천부 패널에 중복 없이 slot set 만 표시.
