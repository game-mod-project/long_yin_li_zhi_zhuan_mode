## Trade-off 문서화

**mod 의 [속성] 탭 자질값 input vs in-game stat panel**:
- mod = raw maxXxx[idx] (stored value, 편집 가능)
- in-game = getter `HeroData.GetMaxXxx(int)` 반환값 (Postfix override 적용)
- 다른 mod (LongYinCheat) 가 자체 cap-bypass Postfix `__result = 999` 반환 → in-game 항상 999 표시
- 본 모드 Phase 3 (`EnableUncapMax = true`) 가 자체 Postfix `__result = UncapMaxAttri` 반환 → 동상
- 따라서 in-game 은 999 / mod 은 raw 333 (예시) — 의도된 차이

- mod = "내가 편집할 수 있는 stored value"
- in-game = "Postfix override 적용된 effective value (다른 mod 또는 본 모드 cap-bypass 영향)"
